# Why open_text_based_rpg?

## Text Based RPG

你知道传统的文字类rpg游戏吗？它们可能是最好上手的rpg，制作起来也最简单。我想，如果需要创作出无限自由度的rpg，可能从文字类rpg入手最简单。

我最早玩的文字类rpg是era系列的一个和东方project有关的作品。里面的地图非常详细，各个地点、人物、物品都有自己的数值和描述等等，交互非常丰富，可以说如果有更丰富的画面将会更加精彩。

## Open

文字类rpg不管怎么说也还是固定的输出，但假如有ai假如就不一样了。为什么不直接用聊天ai玩，而是要立项呢？因为直接和ai对话的时候常常面临长期记忆不准确、数值计算不精确、出现一些不符合设定的幻觉等内容。所以我想要立项，名字叫`open_text_rpg`，用准确的代码存储设定、数值、状态等，用ai生成感性的交互内容，把以前只能用人类来完成的dnd式探索冒险游戏变成可以代码+ai完成的内容。

## Construction

目前预想的结构如下：

```text
OpenTextRPG/
├── 游戏引擎（确定性）
│   ├── 状态管理器 - 本地存储游戏状态
│   ├── 规则引擎 - 处理战斗、技能、经济等规则
│   ├── 事件系统 - 任务触发、世界事件
│   └── 验证器 - 确保AI输出符合游戏规则（也接受手动回滚）
├── AI集成层
│   ├── 上下文构建器 - 组装提示词
│   ├── 输出解析器 - 提取结构化数据
│   ├── 记忆管理器 - 向量数据库存储关键记忆
│   └── 风格控制器 - 保持叙事一致性
└── 交互界面
    ├── Web/移动端前端
    ├── 命令行接口
    └── API服务
```
